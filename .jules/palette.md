## 2025-02-25 - ContentUnavailableView for Empty States
**Learning:** Using 'ContentUnavailableView' (iOS 17+) for empty lists and detail placeholders significantly improves accessibility and provides clear user guidance compared to empty screens or static text. Hiding the 'EditButton' when a list is empty also reduces cognitive load and prevents confusing UI states.
**Action:** Always implement 'ContentUnavailableView' with actionable CTAs for empty collection views in SwiftUI projects.
