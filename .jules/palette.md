## 2025-02-25 - [Standardizing Empty States]
**Learning:** For SwiftUI-based iOS apps (targeting iOS 17+), using `ContentUnavailableView` provides a consistent and accessible way to handle empty states compared to just showing an empty `List`. It guides the user with a clear CTA.
**Action:** Always implement `ContentUnavailableView` when a primary list or detail view is empty.
